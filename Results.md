All numbers below are produced by `POST /evaluate` over the full local dataset of
300 historical tickets. The run is deterministic (fixed seed), so these figures
are reproducible.

## Evaluation snapshot

| Metric | Value | What it measures |
| --- | --- | --- |
| Routing accuracy | 1.00 | Share of tickets whose predicted category matches the labelled category |
| Escalation precision | 0.67 | Of tickets we flag to escalate, how many truly escalated |
| Escalation recall | 0.60 | Of tickets that truly escalated, how many we flagged |
| Escalation F1 | 0.63 | Harmonic mean of precision and recall |
| Escalation ROC-AUC | 0.81 | Ranking quality of the escalation probability |
| Groundedness | 0.74 | Fraction of draft-reply claims supported by a retrieved snippet |
| TTFR saved (proxy) | 1.8 min/ticket | Agent minutes saved per ticket vs. writing the first reply from scratch |
| Cost / 100 tickets | $0.00 | Default offline path uses no paid API |
| Sample size | 300 | Tickets evaluated |

## How each metric is computed

- **Routing accuracy** - exact-match between `classifier.predict` and the ticket's
  labelled category, over all 300 tickets.
- **Escalation precision / recall / F1 / ROC-AUC** - `sklearn.metrics` on the
  boolean escalation decision (threshold 0.5) and the continuous probability.
- **Groundedness** - for a sample of generated drafts, the fraction of content
  words in the draft (excluding boilerplate) that also appear in the retrieved
  evidence snippets. A clarification reply (when retrieval confidence is low)
  counts as fully grounded because it makes no factual claims.
- **TTFR proxy** - a read/write effort model: reading the ticket and retrieved
  context (200 wpm) plus lightly editing a 60-word draft (25 wpm, 25% edit
  fraction) versus writing the whole reply, expressed as minutes saved.
- **Cost / 100 tickets** - token estimate times configured price. With the
  default deterministic generator the price is $0, so the cost is $0. Enabling a
  paid LLM re-prices this line automatically from the `COST_PER_1K_*` settings.

## Honest caveats

- The dataset is synthetic (generated by `scripts/generate_data.py`) and the
  category language is fairly clean, which is why routing accuracy is at ceiling.
  On messy real-world text expect this to land in the 0.85-0.95 range; the
  classifier is intentionally a transparent keyword model so its behaviour is
  auditable.
- Escalation is the harder, more realistic problem. The escalation model combines
  text signals (outages, legal/compliance exposure, urgency, repeated contact)
  with the **customer tier**, since a high-value account with a risk signal is
  more likely to be escalated; this is the "VIP customer + repeated failures"
  case the brief calls out. Adding the tier signal lifted recall and F1 over a
  text-only model. Some residual misses remain because a little label noise is
  injected on purpose to keep the numbers honest rather than perfect.

## What I would do next

- Replace the keyword classifier with a calibrated linear model (TF-IDF +
  logistic regression) once labelled production data is available, keeping the
  same interface so the API does not change.
- Learn the escalation signal weights from data (e.g. logistic regression over
  the same features) instead of hand-tuning them.
- Track groundedness against a held-out human-rated sample to validate the
  automatic proxy.
